\-----アプリケーション仕様書

# 1\. アプリケーション概要

| 項目 | 詳細 | 補足・コンセプト |
| ----- | ----- | ----- |
| **アプリ名** | Gakusyoku Stamp App（仮） |  |
| **インスパイア元** | 「コークオンアプリ」 | 学食の利用頻度に応じたリワードシステムの構築。 |
| **目的** | 学生食堂の利用促進と利用データ（時間帯、メニュー、価格）の収集。 |  |
| **ターゲット** | ○○大学の学生および教職員 |  |
| **形式** | スマートフォン向けWebアプリ (PWA/レスポンシブWeb) | ネイティブアプリ開発のコストを抑える。 |
| **管理方式** | ユーザーごとに利用履歴、スタンプ保有数を管理 |  |

# 2\. 機能要件

## 2.1. ユーザー・認証機能

* **新規登録/ログイン**: 学籍番号（または職員番号）とパスワードによる認証を基本とする。大学のSSO（シングルサインオン）システムとの連携も検討する。  
* **マイページ**: ユーザー情報の確認・編集、現在のスタンプ保有数、獲得可能な特典一覧を表示。  
* **利用履歴**: 食券を読み取った日時、価格、獲得スタンプ数の詳細履歴を表示。

## 2.2. 食券読み取り・スタンプ付与機能

学生食堂で発行された食券をスマートフォンで撮影し、情報をデジタル認識する機能です。

| 項目 | 詳細 | ソース情報 |
| ----- | ----- | ----- |
| **読み取り方法** | スマートフォンのカメラ機能を使用した食券の画像スキャン（OCR併用）。 |  |
| **認識対象情報** | 食券に印字された以下の情報を認識する。 |  |
|  | ・**日付**: YY. MM. DD (左上) |  |
|  | ・**一意な識別子 (ハッシュ値)**: 000000 (右上) |  |
|  | ・**時刻**: HH:mm (左下) |  |
|  | ・**価格**: ￥０００ (右中) |  |
| **スタンプ付与ロジック** | 1\. 食券の読み取りが成功し、不正検知をクリアした場合にスタンプを付与。 |  |
|  | 2\. 基本は食券1枚につき1スタンプを付与する。 |  |
|  | **（追加要素）** 価格に応じたインセンティブ：利用額が500円以上など、特定の金額を超えた場合は2スタンプを付与するなど、利用単価向上策を導入する。 |  |

## 2.3. スタンプ・特典交換機能

* **スタンプ表示**: 現在のスタンプ保有数をグラフィカルなスタンプカード形式で表示。  
* **特典交換**: 貯まったスタンプ数に応じて、アプリ内で特典と交換する。  
  * 例：15スタンプで無料ドリンク券、20スタンプで学食割引クーポンなど。  
  * 交換成立後、特典利用時にレジで提示するQRコードまたはバーコードを発行する。

# 3\. 追加要素の提案

アプリの利用頻度を高め、学生の満足度を向上させるための追加機能を提案します。

## 3.1. 学食情報提供機能

* **本日のメニュー**: 日替わり、週替わりメニュー、限定メニューなどの情報を写真付きで配信。  
* **混雑状況表示**: 食堂内に設置したセンサーや、過去の食券読み取り時間データに基づき、現在の混雑状況（空き、普通、混雑）をリアルタイムで表示し、利用分散を促す。  
* **栄養サマリー**: 読み取った食券のメニュー情報から、その食事で摂取したカロリーや主要栄養素（簡易的）を記録・表示する。

## 3.2. エンゲージメント機能

* **ランク制度**: 利用頻度や累計利用額に応じてユーザーをランク付け（例：ブロンズ、シルバー、ゴールド）。上位ランクのユーザーには、特別メニューの先行購入権や割引率の優遇を提供する。  
* **チャレンジ/ミッション**: 「今週3回学食を利用」「特定のメニューを注文」などのミッションを設定し、達成者にボーナススタンプを付与する。

# 4\. 非機能要件（不正対策の具体化）

不正なスタンプ獲得を防ぎ、公正な利用環境を保つための対策を講じます。

* **重複利用のブロック（必須）**:  
  * 食券に記載された\*\*一意な識別子（ハッシュ値 000000）\*\*をキーとしてサーバー側のデータベースに登録し、一度読み取られたIDが再度読み取られた場合、スタンプ付与をブロックする。  
* **偽造検知（必須）**:  
  * 正規の食券のレイアウト、フォント、印刷品質などの画像パターンを機械学習モデルに学習させ、撮影された画像が正規の食券であるかを高精度で判定し、印刷物やディスプレイ表示された偽造品を検出する。  
* **手書き・不正媒体の読み取り拒否**:  
  * 画像認識（OCR）機能を利用し、食券の情報（日付、価格など）を正確に抽出する。  
  * 特に**手書きテキスト**の特徴を識別する機械学習モデルを導入し、手書きメモや低品質なコピー画像など、不正な媒体からの読み取り要求を自動的に除外する。  
* **時間制限**: 食券に印字された日付と時刻が、読み取り時の日時と比較して一定時間（例：発行から1時間以内、または当日中）を経過していた場合、読み取りを無効とする。
